What TRUEX is (and isn’t) TRUEX verifies records. It does not sell picks. It does not run a feed. It does not allow edits. If it can be edited, it can’t be trusted.

What TRUEX verifies

  • Append-only bet logs (bets can be added, not edited)
  • Lock-time enforcement (no post-start submissions)
  • Frozen snapshots as the only public source of truth
  • Deterministic analytics derived from frozen bets
  • Payload seals (hash changes when any file changes)
  • Strict acceptance tests (fail-fast drift detection)

What TRUEX refuses to be

  • No pick selling
  • No backfilling winners
  • No post-lock edits
  • No silent adjustments
  • No social feed / comments / ranking ladder

How to audit (public)

TRUEX is designed so anyone can audit results without trusting TRUEX. The public site exposes the same artifacts used to compute profiles. If a file changes, the seal changes.

  1. Open the seal: latest/payload_seal.json
  2. Inspect the published artifacts under: latest/ (analytics JSON, verification metadata, receipts)
  3. View a verified profile and click “View Raw Analytics JSON” to confirm the stats are derived from the public artifact.
    Example: /verified/profile/?u=sampleuser
  4. If you want, recompute hashes locally from the published payload and compare to the seal.
